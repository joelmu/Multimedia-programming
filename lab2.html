1. RWD

What is the purpose of RWD? Why it is used? How things are done before RWD? What is the difference?
Responsive web designs purpose is to make website easier to view with different sized devices by using HTML and CSS. Before RWD size of the screen was not thought on the desing because there was not mobile users using the web sites. There was no technology to make it responsive. The website did not look good on smaller devices. The Difference is that before RWD web sites did not scale to smaller devices like they do now.

How css media queries can help you building a responsive web sites?
With css media queries it is possible to make different css rules for different screen sizes.

Why to use em (font) based sizing for margins?
Em is scalable unit and based on current font size. It is good to use em based sizing for example margins because if you then change fotn size by screen size everything changes in right ratio.

Article is 7 years old, do we have any technology which can be used to achieve same type of UI responsiveness as described in the article. Buzz words: (CSS grid, Flexbox)Best technologies for responsive web design are Bootstrap, Foundation, Material UI, Pure and Skeleton. It is also possible to use Dynamically Serving Templates or separate Mobile URLs. DST means that different code will be served to same URL depending what device is used to access the URL.
